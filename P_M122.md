# Portfolio-Eintrag: Automatisierter Server-Ping-Check mit PowerShell

## Einleitung

In unserem Unternehmen müssen jeden Morgen mehrere Server im Netzwerk auf Erreichbarkeit geprüft werden. Früher wurde diese Kontrolle manuell durchgeführt, indem jede IP-Adresse einzeln in ein Ping-Kommando eingetippt wurde. Das ist im Alltag fehleranfällig, weil man sich vertippen kann oder eine Adresse vergisst. Zusätzlich bleibt bei einem manuellen Check ohne Dokumentation kein Nachweis übrig. Wenn später behauptet wird, ein Server sei am Morgen nicht korrekt geprüft worden, kann man das nicht mehr belegen. Nach einem Vorfall, bei dem ein Kunde einen Ausfall meldete und ich im Nachhinein keinen Beweis für meinen Test hatte, war klar: Der Ablauf muss automatisiert werden und die Resultate müssen nachvollziehbar gespeichert sein.

## Umsetzung und Begründung

Ich habe dafür ein PowerShell-Script erstellt, das die Serverliste aus einer separaten Textdatei (`servers.txt`) einliest und danach jeden Eintrag automatisch anpingt. Die Serverliste ist absichtlich ausgelagert, damit bei Änderungen (neue Server, IP-Wechsel) nicht der Code angepasst werden muss, sondern nur die Textdatei. Das Script zeigt pro Server im Terminal klar an, ob er erreichbar ist (grün) oder nicht erreichbar ist (rot). Zusätzlich schreibt es die Resultate in eine Logdatei (`ping.log`). Damit ist jederzeit nachvollziehbar, wann getestet wurde und welcher Status pro Server vorlag.

Technisch wird pro Server genau ein Ping gesendet, weil für die reine Erreichbarkeitsprüfung ein einzelner Ping genügt und der Check dadurch schnell bleibt. Wichtig war ausserdem, dass das Script nicht abbricht, wenn ein Server nicht antwortet oder ein Eintrag problematisch ist. Darum wird bei der Ping-Prüfung ein Fehlerhandling verwendet, sodass der Durchlauf immer fertig wird und alle Server geprüft werden. Das Logfile ist bewusst als einfache Textdatei umgesetzt, damit es ohne zusätzliche Tools direkt lesbar ist und im Streitfall als Nachweis dienen kann.

Der folgende Code ist die umgesetzte Lösung, bewusst einfach gehalten und vollständig kommentiert:

```powershell
###########################################
# Autor: Janis Meier
# Version: 1.0
# Titel: Server-Ping Check mit Logfile
#
# Dateien:
# - servers.txt  (pro Zeile: IP oder Hostname)
# - ping.log     (wird automatisch erstellt / erweitert)
###########################################

# $PSScriptRoot ist der Ordner, in dem dieses Script liegt.
# Dadurch funktioniert das Script unabhängig davon, von wo aus es gestartet wird.
# Die Logdatei wird im gleichen Ordner wie das Script abgelegt.
$LogFile = "$PSScriptRoot\ping.log"

# Endlosschleife:
# Das Script läuft dauerhaft und prüft die Server in regelmässigen Abständen.
# Beenden kann man es z.B. mit CTRL + C im Terminal.
while ($true) {

    # Terminal-Ansicht leeren, damit pro Durchlauf eine frische Ausgabe sichtbar ist.
    # Das beeinflusst nur die Anzeige, nicht das Logfile.
    Clear-Host

    # Serverliste aus der Textdatei einlesen.
    # Jede Zeile in servers.txt wird als eigener Eintrag (IP/Hostname) in $Servers gespeichert.
    $Servers = Get-Content "$PSScriptRoot\servers.txt"

    # Kontrollvariable:
    # Wir starten mit der Annahme, dass alle Server erreichbar sind.
    # Sobald EIN Server nicht erreichbar ist, wird $AllUp auf $false gesetzt.
    $AllUp = $true

    # Zeitstempel für diesen Durchlauf.
    # Dieser wird ins Logfile geschrieben, damit später nachvollziehbar ist, wann geprüft wurde.
    $Time = Get-Date -Format "yyyy-MM-dd HH:mm:ss"

    # Kopfzeile pro Durchlauf ins Logfile schreiben.
    # Add-Content hängt an eine bestehende Datei an (oder erstellt sie automatisch, falls sie nicht existiert).
    Add-Content -Path $LogFile -Value "===== $Time ====="

    # Jeden Server aus der Liste prüfen.
    foreach ($Server in $Servers) {

        # Ping-Prüfung:
        # -ComputerName: Ziel (IP oder Hostname)
        # -Count 1: genau ein Ping pro Server (schnell und reicht für Erreichbarkeit)
        # -Quiet: liefert nur True/False (keine lange Ping-Ausgabe)
        # -ErrorAction SilentlyContinue: Script bricht nicht ab, wenn z.B. ein Hostname nicht auflösbar ist
        $PingOK = Test-Connection -ComputerName $Server -Count 1 -Quiet -ErrorAction SilentlyContinue

        # Wenn Ping erfolgreich war (True) -> Server ist erreichbar.
        if ($PingOK) {

            # Ausgabe im Terminal in Grün (so sieht man sofort: OK).
            Write-Host "$Server ist erreichbar" -ForegroundColor Green

            # Eintrag ins Logfile: Serverstatus als UP dokumentieren.
            Add-Content -Path $LogFile -Value "$Server : UP"

        } else {

            # Ausgabe im Terminal in Rot (auffällig, damit Fehler sofort sichtbar sind).
            Write-Host "$Server ist NICHT erreichbar" -ForegroundColor Red

            # Eintrag ins Logfile: Serverstatus als DOWN dokumentieren.
            Add-Content -Path $LogFile -Value "$Server : DOWN"

            # Mindestens ein Server ist nicht erreichbar -> also sind nicht "alle up".
            $AllUp = $false
        }
    }

    # Nach der Prüfung aller Server:
    # Wenn $AllUp immer noch True ist, bedeutet das: kein einziger Server war DOWN.
    if ($AllUp) {

        # Leerzeile für bessere Lesbarkeit im Terminal.
        Write-Host ""

        # Zusammenfassung im Terminal.
        Write-Host "All servers are UP" -ForegroundColor Green

        # Zusammenfassung auch ins Logfile schreiben (klarer Nachweis).
        Add-Content -Path $LogFile -Value "All servers are UP"
    }

    # Leerzeile im Logfile als optische Trennung zwischen den Durchläufen.
    Add-Content -Path $LogFile -Value ""

    # Pause zwischen den Durchläufen:
    # Ohne Sleep würde das Script permanent pingen und unnötig CPU/Netzwerk belasten.
    Start-Sleep -Seconds 10
}
```

## Setup und automatischer Start

Damit das Script funktioniert, müssen sich `PingCheck.ps1` (oder wie du es nennst) und die Datei `servers.txt` im gleichen Ordner befinden. In der `servers.txt` steht pro Zeile ein Server (IP oder Hostname). Beim ersten Lauf wird die Logdatei `ping.log` automatisch erstellt und danach bei jedem weiteren Durchlauf ergänzt.

Wenn das Script beim Login automatisch starten soll, kann man es über den Windows-Autostart lösen. Dazu öffnet man mit **Win + R** das Ausführen-Fenster, tippt `shell:startup` ein und bestätigt. In diesem Autostart-Ordner erstellt man eine Verknüpfung, die PowerShell mit dem Script startet. Als Ziel kann man z.B. verwenden: `powershell.exe -ExecutionPolicy Bypass -File "C:\Pfad\zu\deinem\PingCheck.ps1"`. So wird das Script beim Anmelden automatisch gestartet, ohne dass man es jeden Morgen manuell öffnen muss.

## Reflexion

Das Script löst das ursprüngliche Problem zuverlässig: Es ersetzt die fehleranfällige manuelle Ping-Arbeit und erstellt gleichzeitig einen nachvollziehbaren Nachweis über die durchgeführten Tests. Besonders das Logfile ist in der Praxis der grösste Gewinn, weil man damit im Störungsfall belegen kann, ob ein Server zu einem bestimmten Zeitpunkt erreichbar war oder nicht. Die Lösung ist bewusst einfach gehalten, damit sie stabil läuft und leicht verständlich bleibt. Als mögliche Verbesserung für später sehe ich vor allem das Thema Logfile-Grösse: Da das Script in einer Schleife läuft, wächst die Datei kontinuierlich. In einer nächsten Version könnte man z.B. alte Einträge automatisch löschen, das Log pro Tag separat führen oder nur Statusänderungen loggen, um die Übersicht weiter zu erhöhen.
